📖 概要

AI Chat Botは、アップロードした情報を基にRAG（Retrieval-Augmented Generation）技術を使用して、より正確で具体的な回答を提供します。

🎯 主な機能
  • 多様な形式での情報アップロード
  • ベクトル検索による高精度な情報取得
  • コンテキストを考慮した回答生成
  • リアルタイムな知識ベース更新

🚀 クイックスタート

1. 最も簡単な方法：手動入力

  1. アップロードページにアクセス
  2. 「手動で情報を入力」セクションを開く
  3. タイトルと内容を入力
  4. 「情報を保存」ボタンをクリック
📝 入力例
タイトル: Pythonの基本文法 内容: Pythonでは、変数の定義に型宣言は不要です。 例: name = "太郎"、age = 25 インデントでコードブロックを表現します。

✏️ 手動入力

最も直接的で確実な方法です。構造化された情報を正確に登録できます。

必須フィールド

  • タイトル: 情報を識別するための見出し
  • 内容: 実際の情報・詳細

任意フィールド

  • カテゴリ: 一般情報、技術情報、ビジネス情報、FAQ、手順・プロセス
  • タグ: カンマ区切りでキーワードを設定
💡 効果的な入力のコツ
  • タイトルは検索しやすい具体的な名前を付ける
  • 内容は構造化して記述する（見出し、箇条書き等）
  • 関連するキーワードをタグに含める

📁 ファイルアップロード

既存の文書ファイルを直接アップロードして知識ベースに追加できます。

アップロード手順

  1. アップロード領域にファイルをドラッグ&ドロップ
  2. または、クリックしてファイル選択ダイアログを開く
  3. 複数ファイルの選択が可能
  4. 「ファイルをアップロード」ボタンでアップロード開始
⚠️ 注意事項
  • ファイルサイズは1ファイル10MB以下を推奨
  • パスワード保護されたファイルは対応していません
  • 画像のみのPDFは認識できない場合があります

🌐 URL取得

WebページのコンテンツをURLから直接取得して知識ベースに追加します。

対応サイト

  • 一般的なWebサイト
  • ブログ記事
  • ニュースサイト
  • 技術文書サイト
🔗 URL例
✅ 良い例: https://qiita.com/example/items/python-basics https://zenn.dev/example/articles/react-tutorial https://docs.python.org/ja/3/tutorial/ ❌ 対応できない例: https://private-site.com/login-required https://pdf-download-link.com/file.pdf

📋 形式詳細

📝 テキストファイル (.txt)

最もシンプルで確実な形式です。UTF-8エンコーディングを推奨します。

サンプル構造
タイトル: Python基礎知識 概要: Pythonは初心者にも優しいプログラミング言語です。 特徴: - シンプルな文法 - 豊富なライブラリ - クロスプラットフォーム対応 基本文法: 変数定義: name = "value" 条件分岐: if condition:

📖 Markdownファイル (.md)

見出し、リスト、コードブロックなどの構造が維持されます。

Markdownサンプル
# API仕様書 ## 概要 RESTful APIの使用方法について説明します。 ### エンドポイント - `GET /api/users` - ユーザー一覧取得 - `POST /api/users` - ユーザー作成 ```python import requests response = requests.get('/api/users') ```

📄 PDFファイル (.pdf)

テキスト情報を含むPDFから内容を抽出します。

📋 制限事項
  • 画像のみのPDFは処理できません
  • 複雑なレイアウトは正しく抽出されない場合があります
  • パスワード保護されたPDFは対応していません

📊 JSONファイル (.json)

構造化されたデータを扱う場合に適しています。

JSON構造例
{ "title": "商品情報", "data": [ { "name": "ノートPC", "price": 80000, "specs": { "cpu": "Intel i7", "memory": "16GB", "storage": "512GB SSD" } } ] }

💡 ベストプラクティス

🎯 効果的な情報構造

  • 明確な見出し: 階層的な見出しで内容を整理
  • キーワードの適切な配置: 検索されやすい単語を含める
  • 具体例の提供: 理論だけでなく実例も含める
  • 関連情報の整理: 関連する項目は近くに配置

📊 推奨ファイル構成

理想的な文書構造
# メインタイトル ## 概要・要約 簡潔な説明 ## 詳細内容 ### サブセクション1 具体的な説明と例 ### サブセクション2 手順や方法 ## まとめ・補足 重要なポイントの再確認
🏆 品質向上のポイント
  • 1つのファイルは1つのトピックに集中
  • 専門用語には説明を併記
  • 例文・サンプルコードを豊富に含める
  • 定期的な内容の見直しと更新

🏷️ メタデータ

アップロードした情報には自動的にメタデータが付与され、検索精度の向上に役立ちます。

自動付与される情報

  • アップロード日時
  • ファイル形式
  • 文書の長さ
  • 推定される主要キーワード

ユーザー指定可能な情報

  • カテゴリ分類
  • タグ（キーワード）
  • 重要度レベル
  • 更新頻度

🔧 トラブルシューティング

よくある問題と解決方法

❓ ファイルがアップロードできない

  • ファイルサイズが10MBを超えていないか確認
  • 対応形式（.txt, .pdf, .docx, .md, .json）であるか確認
  • ファイルが破損していないか確認
  • ブラウザを再起動して再試行

❓ PDFの内容が正しく認識されない

  • 画像のみのPDFではないか確認
  • テキスト選択可能なPDFであるか確認
  • パスワード保護を解除してから再アップロード
  • 可能であればテキスト形式で保存し直す

❓ URLから情報を取得できない

  • URLが正しく入力されているか確認
  • ログインが必要なページでないか確認
  • JavaScriptで動的生成されるコンテンツは取得できません
  • ページの内容をコピーして手動入力を試行

❓ 検索で期待した結果が出ない

  • キーワードを変えて検索してみる
  • 類似する表現や同義語で試行
  • 情報のタイトルや見出しを確認
  • 情報が正しくアップロードされているか確認
🆘 サポートが必要な場合

上記で解決しない問題がある場合は、以下の情報を含めてお問い合わせください：

  • 発生した問題の詳細
  • エラーメッセージ（ある場合）
  • 使用しているブラウザとバージョン
  • アップロードしようとしたファイルの種類とサイズ
